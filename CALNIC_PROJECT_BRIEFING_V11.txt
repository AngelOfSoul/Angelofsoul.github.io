WHAT IS THIS SITE?
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Calnic Online is a digital memory book ("cartea vie a satului") for
Calnic, a village in Alba County, Transylvania, Romania.

Each family from the village can create their own private page,
add family members, a family history, a timeline of important events,
and upload photos. The site is fully bilingual: Romanian + English.

  GitHub repo:  https://github.com/AngelOfSoul/Angelofsoul.github.io
  Live site:    https://angelofsoul.github.io  (temporary URL)
  Final domain: https://calniconline.ro         (domain not yet purchased)
  Upload link:  https://github.com/AngelOfSoul/Angelofsoul.github.io/upload/main

ALL PAGES ON THE SITE (13 total)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Every page has a RO/EN language button in the top-right corner.
The user's language choice is remembered across all pages automatically.

  1. index.html             — Home page
                              Welcome, mission, site overview, how to join.

  2. istorie.html           — Village History
                              Timeline of Calnic's history from ancient times
                              to today. Photos, facts, "Did you know?" cards.

  3. familiile.html         — Families Directory
                              Lists all registered families. Search by name
                              or filter by first letter. Loads from database.

  4. familiile-familie.html — Individual Family Page
                              A family's public profile: story, members,
                              timeline, photos. PIN-protected private section.
                              Opened via: familiile-familie.html?family=ID

  5. genealogie.html        — Genealogy Overview
                              Search for a family to see their family tree.

  6. genealogie-familie.html— Family Tree Viewer
                              Interactive SVG tree showing all family members
                              across generations with bio editing.
                              Opened via: genealogie-familie.html?family=ID

  7. galerie.html           — Photo Gallery
                              Village photos sorted by category and year.
                              Logged-in families can upload and compress photos.

  8. harta.html             — Village Map
                              Interactive map (Leaflet.js) showing Calnic's
                              landmarks: castle, church, key locations.

  9. login.html             — Family Account
                              Sign in or register a new family account.
                              Step 2 of registration: family name, year,
                              description, members, timeline, PIN setup.

 10. dashboard.html         — Private Family Dashboard
                              Only visible after login. Manage family story,
                              members, timeline, PIN, privacy settings,
                              and MY PHOTOS (new in V11).

 11. contact.html           — Contact Form
                              Send a message, report a bug, or ask a question.
                              Messages go to the site owner via Formspree.

 12. intrebari.html         — FAQ (Frequently Asked Questions)
                              Plain-language answers about how the site works,
                              privacy, genealogy, and how to add your family.

 13. 404.html               — Page Not Found
                              Friendly error page when a wrong URL is visited.

[ADMIN] TASK A — Set up the photos table in Supabase
─────────────────────────────────────────────────────────────────────
PRIORITY: HIGH — without this, photo uploads will fail

What it is: The photos database table and storage bucket need to be
created manually in Supabase. The SQL file is ready.

Steps:
  1. Go to https://supabase.com → your project → SQL Editor
  2. Click "New query"
  3. Copy the contents of setup-photos-storage.sql from this repo
  4. Paste it into the SQL editor and click "Run"
  5. Then go to Storage (left sidebar) → create a bucket named: photos
  6. Set it to PUBLIC
  7. Add policy: SELECT = anyone can view (true)
  8. Add policy: INSERT = only logged-in users (auth.uid() IS NOT NULL)

Full guide: see README-SUPABASE.md, Step 8

[ADMIN] TASK D — Connect calniconline.ro domain to the site
─────────────────────────────────────────────────────────────────────
PRIORITY: MEDIUM — do this after buying the domain (Task C)

What "DNS" means: DNS is like a phone book for the internet. You need
to tell the domain "when someone visits calniconline.ro, send them
to GitHub Pages where the site is hosted."

Steps:
  1. Log in to your domain registrar (where you bought it in Task C)
  2. Find DNS settings or "Manage DNS records"
  3. Add a CNAME record:
       Name:  www
       Value: angelofsoul.github.io
       TTL:   3600 (or "automatic")
  4. For the root domain (without www), add these 4 A records:
       Name: @ (or leave blank — means root)
       Values (add one record for each IP):
         185.199.108.153
         185.199.109.153
         185.199.110.153
         185.199.111.153
  5. Wait 24-48 hours for DNS to propagate worldwide
  6. Then go to your GitHub repo → Settings → Pages
     → Custom domain → type: calniconline.ro → Save
     → Tick "Enforce HTTPS" (may take a few minutes to activate)

Note: The CNAME file in this repo already contains "calniconline.ro"
which is the GitHub side of this setup. Step 6 above just confirms it.

DATABASE TABLES
  families   — one row per registered family
    id, owner_id, name, village, since, desc_ro, desc_en,
    members_count, generations_count, photos_count,
    show_members, show_generations, show_photos, show_since,
    has_public_photos, has_private_photos, pin_hash, created_at

  members    — family members (children of families)
    id, family_id, name, initial, role, birth_year, death_year, is_deceased

  timeline   — historical events per family
    id, family_id, year, text_ro, text_en

  photos     — uploaded photo metadata (NEW in V10)
    id, family_id, uploader_id, path, title_ro, title_en,
    caption_ro, caption_en, dedicatie_ro, dedicatie_en,
    year, category, is_private, uploaded_at

SUPABASE STORAGE BUCKET
  Name:    photos
  Access:  PUBLIC (read without auth)
  Policies:
    SELECT: true (anyone can view)
    INSERT: auth.uid() IS NOT NULL (only logged-in users)
  Path:    {userId|'public'}/{timestamp}.jpg

HELPER RPC FUNCTION

  Compares encode64(p_pin) against stored pin_hash in families table
