WHAT IS THIS PROJECT?
─────────────────────
A digital village memory book ("cartea vie a satului") for Calnic,
a village in Alba County, Transylvania, Romania. Each family can have
a public profile and a private dashboard space. Goal: preserve
memories, genealogy and photos for future generations.

Bilingual: Romanian (primary) + English (secondary), RO/EN toggle.
Always treat the site as fully live and functional.

BACKEND — FINAL DECISIONS
──────────────────────────
Backend:      Supabase (auth, database, photo storage)
Keep-alive:   cron-job.org (ping every 3 days)
Contact form: Formspree — endpoint: https://formspree.io/f/xpqygdjn
Photos:       Stored compressed in Supabase (NOT Cloudinary)
Security:     Row-level security — families see only their own data

PIN SYSTEM:
- Family chooses their own PIN (4-8 digits, numeric only)
- PIN stored HASHED in Supabase as btoa(pin) — nobody can see it
- Admin cannot see it; can only trigger reset link to family email
- MUST show on familiile-familie.html:
  RO: "PIN-ul este cunoscut doar de membrii familiei.
       Administratorul site-ului nu are acces la continutul privat."
  EN: "The PIN is known only by family members.
       The site administrator has no access to private content."
- Validation via Supabase RPC: check_family_pin(family_id, pin)

NOT chosen: Firebase (risk of shutdown)

  show_photos       boolean
  show_since        boolean

  has_private_photos boolean
  pin_hash          text (base64 encoded via btoa())
  created_at        timestamptz

members
  id          uuid PK
  family_id   uuid FK -> families (cascade delete)
  name        text
  initial     text
  role        text
  birth_year  integer
  death_year  integer
  is_deceased boolean

timeline
  id         uuid PK
  family_id  uuid FK -> families (cascade delete)
  year       integer
  text_ro    text
  text_en    text

photos  (NEW in V10 — run setup-photos-storage.sql to create)
  id           uuid PK (auto)
  family_id    uuid FK -> families (nullable — village photos)
  uploader_id  uuid FK -> auth.users (nullable)
  path         text (Supabase Storage path in 'photos' bucket)
  title_ro     text
  title_en     text
  caption_ro   text
  caption_en   text
  dedicatie_ro text
  dedicatie_en text
  year         integer
  category     text ('cetatea','biserica','viata','evenimente','natura','vechi','familii')
  is_private   boolean default false
  uploaded_at  timestamptz default now()

ROW LEVEL SECURITY — all active:
- Anyone can READ families, members, timeline, public photos
- Only owner_id user can WRITE/UPDATE their own family row
- Only family owner can INSERT/DELETE their own members + timeline
- Any authenticated user can INSERT photos
- Only uploader can UPDATE/DELETE their own photos

STORAGE BUCKET — photos (NEW in V10):
  Name:        photos
  Visibility:  PUBLIC (read without auth headers)
  Policies:    SELECT = true (anyone) | INSERT = auth.uid() IS NOT NULL
  Path format: {userId|'public'}/{timestamp}.jpg

────────────────────────────────────────

Photo object structure (used by all rendering functions):
  {
    id:          integer (index in current photos array)
    dbId:        uuid    (Supabase row id, only present for DB photos)
    titleRo:     string
    titleEn:     string
    cat:         string  (one of the 7 category ids)
    year:        integer (0 if unknown)
    byRo:        string  (empty for DB photos — uploader name not stored)
    byEn:        string
    captionRo:   string
    captionEn:   string
    dedicatieRo: string
    dedicatieEn: string
    priv:        boolean
    addedOrder:  integer (used for "recently added" sort)
    featured:    boolean (first photo in DB = featured)
    imgUrl:      string|null (null for demo photos, real URL for DB photos)
  }

Rendering rules:
  imgUrl present -> <img src=imgUrl> element
  imgUrl absent  -> SVG_SHAPES[id % SVG_SHAPES.length] placeholder

Image compression in savePhoto():
  Max dimension: 1200px (width or height, preserves aspect ratio)
  Format: JPEG, quality 0.82
  Implementation: FileReader -> Image -> canvas.toBlob('image/jpeg', 0.82)

Upload path format: {userId|'public'}/{Date.now()}.jpg

supabase:ready listener: loadPhotosFromSupabase()
  - Queries: photos WHERE is_private=false ORDER BY uploaded_at DESC
  - On error or empty result: keeps demo data (no visible change)
  - On success: replaces photos array, calls setLang(currentLang)

LANGUAGE PERSISTENCE PATTERN (critical — use everywhere)
──────────────────────────────────────────────────────────
localStorage key: 'calnic-lang'   <-- NEVER use 'lang' alone

Standard pattern for ALL pages:

  var currentLang = localStorage.getItem('calnic-lang') || 'ro';

  function setLang(lang) {
    currentLang = lang;
    localStorage.setItem('calnic-lang', lang);
    document.getElementById('btn-ro').classList.toggle('active', lang === 'ro');
    document.getElementById('btn-en').classList.toggle('active', lang === 'en');
    document.querySelectorAll('[data-ro]').forEach(function(el) {
      var val = el.getAttribute('data-' + lang);
      if (val !== null) el.textContent = val;
    });
    // ... page-specific re-renders
  }

  document.getElementById('btn-ro').addEventListener('click', function() { setLang('ro'); });
  document.getElementById('btn-en').addEventListener('click', function() { setLang('en'); });
  setLang(currentLang);

NOTE — istorie.html uses curLang (not currentLang) and setLangFull:
  var curLang = localStorage.getItem('calnic-lang') || 'ro';
  function setLangFull(lang) {
    curLang = lang;
    localStorage.setItem('calnic-lang', lang);
    setLang(lang);
    showFact(curFact, lang);
  }
  setLangFull(curLang);
